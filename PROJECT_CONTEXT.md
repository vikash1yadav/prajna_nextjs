## ⚙️ Backend Architecture & Class-Based Design Patterns

To maintain a clean enterprise structure without TypeScript, the backend codebase must strictly follow a **Layered Object-Oriented Architecture using ES6 Classes**:

1.  **Controller Layer (`class DashboardController`)**
    * Responsible purely for parsing HTTP requests, managing request payloads, and executing matching methods.
    * Methods must accept `req`, `res`, and `next` parameters, using structured `try/catch` enclosures.
    * Controllers must hand off all database logic to the Service Layer.
2.  **Service Layer (`class DashboardService`)**
    * Responsible for carrying out core business logic calculations, permissions evaluations, and processing complex rules.
    * Accesses the database strictly by interfacing with the Repository Layer.
3.  **Repository Layer (`class DashboardRepository`)**
    * Responsible entirely for low-level database operations using Sequelize models.
    * This abstracts raw Sequelize syntax completely away from controllers and services.
4.  **Utility Layer (`class FormatUtil` / `class CryptoUtil`)**
    * Shared algorithmic functions or format calculations must be isolated inside standalone utility classes using static methods where appropriate.

### Example Layer Structuring Pattern:
```javascript
// Example of the required Class-Based pattern format:
import AttendanceRepository from '../repositories/attendanceRepository.js';

class AttendanceService {
    constructor() {

        // Business logic here

    }
}
